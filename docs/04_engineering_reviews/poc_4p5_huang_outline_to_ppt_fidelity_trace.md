## 1. Executive verdict
"Outline is correct but the final PPT doesn't match" is **by design, not a one-off bug**: the per-slide
content is **regenerated by the LLM** from the outline as a *hint*, then mapped into template shapes.
Only three things are currently hard-enforced end-to-end: (a) the **template page** a slide maps to
(`template_page_number`), (b) the **section/topic binding** for body slides ("目录归属（必须遵守）"), and
(c) the **agenda item slots** (P1 deterministic fill). Everything else — **cover title/subtitle, body
title, body points** — is an LLM-regenerated *hint*, and the **cover has no deterministic fill at all**.
So: agenda is now fixed for the slots that exist; **cover mismatch is the real open gap** (no targeted
fill + template-dependent slot mapping + LLM rephrase). The safe path is **verification gate (option 1)
+ a cover-only deterministic fill mirroring P1 (option 2)**; never force body titles/points (that is what
made Q3d regress).

## 4. Where outline can be overwritten or ignored
- **`_cards_to_slides`** copies cards verbatim → `confirmed_outline.json` is faithful (no overwrite here).
- **`generate_slide_content`** REGENERATES `title`/`subtitle`/`modules`/`table_data` via the LLM
  (`content_generator.py` slide schema asks for `"title": "<slide title…>"`). The outline title/points are
  prompt context, not copied through → **the rendered title/points are the LLM's, not the user's verbatim**.
- **`content_normalizer`** truncates/caps modules to slot capacity → long points are shortened.
- **`_fill_with_blueprint`** writes the (LLM) title into the blueprint's **title slot**; if the analyzer
  mapped the title slot to a **non-visible/wrong shape**, the visible title shape keeps **template residue**.
- **`deck_polish`** (P1) overwrites only the **agenda** content slots; **cover has no equivalent**, and the
  residue rectangles on cover/agenda are not cleaned (no global cleaner — deliberately, post-Q3d).

## 5. Cover mismatch root cause
Three compounding causes, none currently mitigated:
1. **No deterministic cover fill.** Unlike agenda (P1), nothing forces the outline cover title/subtitle
   into the cover's visible shapes after render. (`fix_cover_slide` existed in Q3d but was reverted with it.)
2. **LLM regenerates the cover title** (outline title is a hint). It may rephrase or drop the user's exact text.
3. **Analyzer cover title-slot mapping is template-dependent.** On local template 28 the title slot maps to
   the visible `标题 3`; in the Q3b job_104 era the cover blueprint mapped the title to a non-visible
   `文本框 1`, so `_fill_with_blueprint` wrote a hidden shape and the visible `标题 3` kept residue
   (`央国企穿透式监管…`). On the cloud t5/t5_new master the cover likely has multiple/overlapping text boxes →
   the visible title is not the mapped slot → residue persists regardless of a correct outline.
→ **Answer A:** it's NOT the frontend (cards carry the title) and NOT the save (confirmed_outline is
faithful). It's (2)+(3)+(1): LLM rephrase + wrong/!visible slot mapping + **no cover deterministic fill +
template residue shape not overwritten**.

## 6. Agenda mismatch root cause
- The outline agenda items are correct; `_agenda_consistency` derives the deduped body section_titles;
  **P1 `fill_agenda_deterministic` now writes those exact strings into the agenda content-slot shapes**
  (verified: job 123/125 agenda items == exact section_titles, closing excluded).
- Remaining agenda imperfections are **template-side**, not outline/fill bugs:
  1. The T5 agenda page has **only 3 item slots** but decks produce **5–6 sections** → only first 3 show
     (logged `agenda_overflow`).
  2. **Extra template text** on the agenda slide (`国资委穿透式监管解读`, `落地路径及相关保障`, `案例实践`,
     numbering labels) lives in shapes that are **not agenda content slots**, so P1 (correctly scoped to the
     content slots) doesn't touch them, and there is no residue cleaner → they remain as "多余注入".
  3. Visual top-to-bottom order of filled items follows template slot geometry (shape order), not section order.
→ **Answer B:** NOT "LLM rephrase" anymore (P1 overwrites). It's (template 3 slots) + (extra template
shapes not covered by slot_refs) + (no residue clean). agenda_slot_refs is complete **for the content
slots**, but the agenda page has decorative/residue shapes outside those slots.

## 7. Body / table pages contract analysis
- **Body title/points are HINTS.** `generate_slide_content` regenerates them; `content_normalizer` caps
  them. The **hard constraint is the section/topic binding** ("目录归属（必须遵守）" from `section_title` +
  `slide_role_under_section`), which keeps the body page on-topic — this is why job_28/job_37 body pages are
  acceptable. Forcing verbatim body titles/points would fight the capacity normalizer and risk regressions.
- **Table pages:** when a page has a table, the slide prompt instructs the LLM to return `table_data`
  `{headers, rows}` and **feeds it the template's headers** (`content_generator.py:489` `表格表头为: {headers}`).
  The template's t5 page-8/9 headers are CMCC residue `['模块层级','分析场景','场景说明','是否模型','分析场景']`,
  so the LLM **echoes the residue** into `table_data.headers` → `_fill_table` writes them. Thinness =
  `content_normalizer` caps rows to `template_rows-1` + `_truncate_cell_text` per column. Both are template +
  capacity driven (do NOT "fix" via truncation changes — high regression risk).

## 8. Why Q3d regressed and how to avoid repeating it
- Q3d bundled four things: deterministic agenda fill (good), **targeted cover fill** (good idea),
  **a GLOBAL `clear_known_residue`** that blanked ANY shape/table cell containing a residue token, and a
  `chat_structured` JSON-fence/retry change. The **global residue cleaner blanked legitimate body/table
  content** (P4 weakened, P8/P9 cells emptied) → content regression. The agenda/cover **targeted** fills were
  not the cause (P1 re-introduced the agenda fill alone with no regression — P0 smoke PASS).
- **Avoidance rule:** any cover/agenda/residue work must be **targeted to the specific shapes** (cover title
  shape; agenda content slots; table header cells only) and **never a deck-wide blanking pass**, and must not
  change `chat_structured`/content prompts. Gate every change with the P0 content smoke.

## 9. Recommended minimal repair plan (3 options, by risk)
**Option 1 — Read-only trace + verification gate (LOWEST risk; recommended first).**
Extend `scripts/poc_content_baseline_smoke.py` (or a sibling) with read-only fidelity checks against
`confirmed_outline.json`: `cover_title_match`, `agenda_items_match` (first N slots == deduped section_titles),
`body_section_title_present`, `table_cell_count`, `P4_text_count`. No generation change. Quantifies the gap
and gates future fixes.

**Option 2 — Special-page deterministic fill (LOW–MEDIUM; recommended next).**
Add a **cover-only** deterministic fill in `deck_polish` mirroring P1 (write `confirmed_outline` cover
`title` into the cover's title shape — prefer name `标题/title`, else top shape; subtitle into a 副标题 shape).
Agenda already done (P1). **No body, no content_generator, no normalizer, no table fill, no global cleaner.**
Optionally a **table-header-cell-only** residue replace (P2, separate). Gate with P0 smoke.

**Option 3 — Outline contract enforcement (HIGH; not recommended for body).**
Persist `effective_outline.json` and force confirmed == effective == final for cover/agenda only (safe);
extending it to body titles/points would override the LLM/normalizer and risks the Q3d-class regression →
**do not enforce body verbatim.**

Run before/after any fidelity change against a freshly generated 12-page T5:
- `cover_title_match` — cover visible title shape text == confirmed_outline cover title.
- `agenda_items_match` — agenda content-slot texts == first N deduped body section_titles (N = slots).
- `body_section_title_match` — each body slide's injected section binding ∈ agenda items.
- `table_cell_count` — P8/P9 non-empty cells ≥ baseline (P0).
- `P4_text_count` — P4 text chars ≥ baseline (P0).
PASS/WARN/FAIL; body metrics (table_cell_count, P4_text_count) are **hard gates** (FAIL blocks); cover/agenda
matches start as WARN until Option 2 lands, then become gates.
